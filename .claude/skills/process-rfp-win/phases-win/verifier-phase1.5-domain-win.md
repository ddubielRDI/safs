# Verifier — Phase 1.5 Domain Detection

## When this runs

After phase1.5-domain-win reports done, BEFORE phase1.6-evaluation-win is invoked. This is the first jurisdiction-anchor checkpoint — every downstream Stage 1 phase inherits its state/agency/statute claims, so a faulty anchor here poisons the entire pipeline (MARS RFP incident, 2026-05-20).

## Inputs (read in this order)

1. `{folder}/shared/domain-context.json` — primary output under verification
2. `{folder}/flattened/*.md` — the actual RFP text (required for grep-verifying every jurisdiction citation)
3. `{folder}/flattened/_combined.md` (if present) — used to confirm citation file:line accuracy

## Verification Checks

### Check 1 — File exists and is valid JSON, structurally complete

**Criterion:** `shared/domain-context.json` exists AND parses as valid JSON AND contains top-level keys: `detected_at`, `selected_domain`, `confidence`, `clear_winner`, `margin`, `scores`, `profile`, `custom_terminology`, `jurisdiction_anchor`. File size >= 200 bytes.

**Pass:** All keys present, JSON parses, size threshold met.

**Fail:** File absent, JSON parse error, or any top-level key missing.

**Evidence to cite:** Specific missing key path (e.g., `jurisdiction_anchor` absent) or parse-error line number.

**Hard-rule reminder:** NEVER claim the file is missing without `Glob` verification first.

---

### Check 2 — `jurisdiction_anchor` present with full sub-key schema (J1)

**Criterion:** `jurisdiction_anchor` is a dict containing sub-keys: `state`, `address_token`, `procurement_portal`, `statute_citation`, `issuing_agency_line`, `solicitation_number`, `primary_rfp_file`, `extraction_method`. The `extraction_method` value MUST contain the substring `"grep_cite"` (proves anchor extraction ran, did not free-style).

**Pass:** All 8 sub-keys present AND `extraction_method` contains `grep_cite`.

**Fail:** Any sub-key missing OR `extraction_method` lacks `grep_cite` token (indicates LLM inference, not grep extraction).

**Evidence to cite:** List the actual sub-keys found vs the required set; print the `extraction_method` value verbatim.

---

### Check 3 — Every non-null anchor field has a `citation` of form `{file}:{line}` (J2)

**Criterion:** For each of `state`, `address_token`, `procurement_portal`, `statute_citation`, `issuing_agency_line`, `solicitation_number`: if the field's `value` is non-null, its `citation` sibling MUST match the regex `^[^:]+\.md:\d+$` (filename ending `.md`, colon, line number). Null `value` requires null `citation` (no orphan citations).

**Pass:** All populated fields have valid `{file}:{line}` citations; null fields have null citations.

**Fail:** Any non-null value lacks a citation OR has a malformed citation (no colon, no `.md` filename, non-numeric line).

**Evidence to cite:** Field name, value, citation actual vs required pattern. List up to 5 offending fields.

---

### Check 4 — Citations are grep-verifiable in the cited file (J3)

**Criterion:** Pick at least 3 non-null populated anchor fields. For each, open `flattened/{file}` from the citation, grep for the `evidence_text` (or `value` if no evidence_text), and confirm the match occurs within ±5 lines of the cited line number. This proves the citation was generated by actual extraction, not fabricated.

**Pass:** All sampled fields grep-verify at the claimed line ±5.

**Concern:** 1 of 3 sampled fields misses by more than 5 lines but finds the text elsewhere in the same file. Log advisory.

**Fail:** Any sampled field's `evidence_text` is not present anywhere in the cited file (hallucinated citation — MARS incident pattern).

**Evidence to cite:** For each sampled field: cited line, grep-found line, delta. Paste the actual grep result snippet.

---

### Check 5 — Statute prefix matches state (J4 — anti-hallucination)

**Criterion:** If `jurisdiction_anchor.state.value` is non-null AND `jurisdiction_anchor.statute_citation.value` is non-null, the statute string MUST start with a prefix consistent with the state. Allowed mappings (extend deterministically as new states surface):
- Oregon → must start with `ORS` or `OAR`
- Washington → must start with `RCW` or `WAC`
- California → must start with `Cal.`
- Texas → must start with `Tex.`
- Alaska → must start with `AS`
- New York → must start with `N.Y.`
- Florida → must start with `Fla.`
- Federal-only (no state match) is acceptable: `\d+ CFR` or `\d+ U.S.C.`

If the statute prefix does NOT match the state, the phase should have nulled it out with `dropped_reason` populated. A non-null contradictory statute is a FAIL.

**Pass:** State+statute combinations match allowed prefix OR statute is null OR `dropped_reason` is populated.

**Fail:** State="Oregon" with statute starting "AS Title" / "RCW" / any non-ORS/OAR prefix (MARS incident: Oregon RFP shipped with Alaska statute numbers).

**Evidence to cite:** `state.value`, `statute_citation.value`, expected prefix list, mismatch description.

---

### Check 6 — `confidence` is a float between 0.0 and 1.0

**Criterion:** `confidence` value is numeric (int or float), >= 0.0, <= 1.0.

**Pass:** Value is in `[0.0, 1.0]`.

**Fail:** Value is null, negative, > 1.0, or non-numeric.

**Evidence to cite:** Actual `confidence` value and its Python `type()`.

---

### Check 7 — All DOMAIN_SIGNALS domains scored

**Criterion:** `scores` dict contains entries for at least these 6 expected domains: `state_local_government`, `education`, `healthcare`, `finance`, `government`, `ecommerce`. Each entry must have keys `score` (numeric) and `top_matches` (dict, possibly empty).

**Pass:** All 6 domains present with proper schema; top 3 retained (phase only stores top 3 — that's correct, but verify the winner appears).

**Fail:** Fewer than the expected domains evaluated (indicates scoring loop was short-circuited), OR `scores` is empty/missing.

**Evidence to cite:** List actual domain keys found in `scores` vs expected set.

---

### Check 8 — `selected_domain` consistent with score + margin

**Criterion:** If `selected_domain` != `"default"`, then `selected_domain` MUST equal the domain with the highest score in `scores`. The phase's selection gate accepts EITHER `(clear_winner AND confidence >= 0.5)` OR `(margin >= 30 AND top_score >= 60)`. Verify the selection respects one of these paths.

**Pass:** `selected_domain` is the top-scored domain OR is `"default"` because no gate was met.

**Fail:** `selected_domain` is not the top-scored domain AND is not `"default"` (silent override or LLM-injected pick).

**Evidence to cite:** Top domain by score, `selected_domain` value, margin, confidence.

---

### Check 9 — CMMC flag has both defense word AND corroborator (anti-false-positive)

**Criterion:** If `cmmc_flag.detected == true`, the underlying RFP text MUST contain at least one defense word (`\bDoD\b`, `\bdefense\b`, `\bmilitary\b`, etc. with word boundaries) AND at least one DoD corroborator (`DFARS`, `Department of Defense`, `armed forces`, `CMMC`, `DoD contract`, `defense contractor`). The MARS run shipped a false-positive cmmc_flag because "defense of claims" (indemnity boilerplate) triggered substring matching — word-boundary + corroborator is the fix.

**Pass:** `cmmc_flag` absent, OR `cmmc_flag.detected == true` AND grep verifies BOTH a defense-word AND a corroborator term in flattened text.

**Fail:** `cmmc_flag.detected == true` with no corroborator present in the RFP text (legal-defense phrasing or other false-positive trigger).

**Evidence to cite:** Grep results for defense words AND corroborator hits in the flattened content (or the absence of corroborators).

---

### Check 10 — No `*_domain_hints` blocks (J5)

**Criterion:** `domain-context.json` MUST NOT contain any top-level or nested key matching the pattern `*_domain_hints` (e.g., `mars_domain_hints`, `{rfp}_domain_hints`). Per-RFP hint dictionaries are a known footgun that mix inferred industry tagging with hallucinated jurisdiction.

**Pass:** No `_domain_hints` keys anywhere in the JSON tree.

**Fail:** Any `*_domain_hints` key found.

**Evidence to cite:** Path to the offending key + a snippet of its content.

---

### Check 11 — Encoding integrity

**Criterion:** `domain-context.json` round-trips cleanly through UTF-8 read/write — no mojibake sequences (`â€"`, `Ã©`, `â€™`, etc.) in any string value.

**Pass:** No mojibake byte sequences in any value.

**Fail:** Any mojibake sequence found in evidence_text, value, or rationale strings.

**Evidence to cite:** Path to the offending string + the mojibake byte sequence.

---

## Disposition Logic

- **PASS:** All of Checks 1, 2, 3, 4, 5, 6, 10, 11 pass AND Checks 7, 8, 9 pass.
- **CONCERN:** Check 4 hits the advisory band (1 sample off by >5 lines but found elsewhere in same file). Log + continue to Phase 1.6.
- **FAIL:** Any of Checks 1, 2, 3, 5, 10, 11 fail OR Check 4 fails on any sample OR Checks 7, 8, 9 fail.

## Self-Test Cases

**Known-bad input (MARS incident replica):**

```
domain-context.json scenario: selected_domain="state_local_government" (correct).
  jurisdiction_anchor.state.value="Oregon" (citation "RFP.md:24" verifies).
  jurisdiction_anchor.statute_citation.value="AS Title 36" (cited "RFP.md:156" but
    grep shows "Oregon Revised Statutes 297.405" at line 156, not "AS Title 36").
  jurisdiction_anchor.issuing_agency_line.value="State of Alaska, Department of
    Administration, Division of Finance" (no citation found in flattened text).
  Top-level extra key "mars_domain_hints" present with hand-tuned Alaska tags.
  cmmc_flag.detected=true; flattened text contains "defense of claims" but no
    DFARS/Department of Defense/CMMC tokens.
```

Verifier MUST detect: Check 3 (issuing_agency has no citation), Check 4 (grep does not verify), Check 5 (Oregon state with AS prefix), Check 9 (cmmc without corroborator), Check 10 (mars_domain_hints present). Disposition: FAIL.

**Known-good input:**

```
domain-context.json scenario:
  selected_domain="state_local_government", confidence=0.42, margin=113, top_score=230.
  jurisdiction_anchor.state.value="Oregon", citation="RFP.md:24",
    evidence_text="State of Oregon" (grep-verified at line 24).
  jurisdiction_anchor.statute_citation.value="ORS 297.405", citation="RFP.md:156"
    (grep-verified, prefix matches Oregon).
  jurisdiction_anchor.procurement_portal.value="OregonBuys", citation="RFP.md:9".
  cmmc_flag absent (no DoD signals in text).
  No *_domain_hints blocks.
  All 6 domains scored.
```

Verifier MUST PASS all checks. Disposition: PASS.

## Hard Rules

1. NEVER claim domain-context.json is missing without `Glob` verification first.
2. NEVER mark a citation as "verified" without actually grep-reading the cited file at the cited line.
3. NEVER accept a non-null statute that doesn't match its state's prefix — even if the citation grep-verifies; the statute itself is the integrity claim.
4. Every finding must cite a specific field path + value (e.g., `jurisdiction_anchor.statute_citation.value = "AS Title 36"` with state="Oregon").
5. On FAIL, return corrective instructions with the specific numeric/textual discrepancy so the phase agent can target the exact repair without re-running everything.
6. NEVER suggest "just delete the jurisdiction_anchor and proceed" — the anchor is the canonical source for all downstream phases; integrity here is non-negotiable.
